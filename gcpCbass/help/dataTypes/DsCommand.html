The DsCommand datatype.
DsCommand

The command to send to the thermometry task.

Input Format:

An option from:

  • address - Address selected device. This option requires that the address argument also be present.
  • checksum - Add checksumming to data written to and read from the temperature sensors.
  • nochecksum - Remove checksumming from data written to and read from the temperature sensors.
  • search - Search for devices attached to its bus and record the ROM id of each device found. This will sequentially fill the register thermo.romid with the ROM id of detected devices; the register thermo.temp will subsequently record the temperature of the devices in the same order. To associate a device ROM id with a particular index of the thermo registers, use the romid option.
  • display - Display the list of known devices.
  • read - Read the temperature of a selected device. This option requires that the address argument also be present. If the specified device is currently flagged as unreachable, the thermometry task will unflag it and attempt to read from it.
  • readall - Read the temperature of all known devices. Any devices flagged as unreachable will not be read.
  • reset - In theory, this causes the DS2480 to initiate a master reset cycle, equivalent to power up.
  • romid - Associate a specific device with an index of the thermo.temp register. This option requires that the address and index arguments also be present.
  • flag - Flag the specified device as unreachable. This option requires that the address argument also be present. When a device is flagged as unreachable, the thermometry task will not attempt to read from that device, even when the readout interval is non-zero.
  • unflag - Flag the specified device as reachable. This option requires that the address argument also be present.
