The Register List.

A single frame of data is divided into a three-tiered hierarchy of related data items which I refer to as "registers" in this document. At the lowest level, a register consists of a block of data of arbitrary type, organized into a multi-dimensional array. These blocks are grouped into logically-related units called "boards", typically representing data from a discrete hardware unit (a CAN module, for example) or a logically distinct operation (for example tracking, as opposed to receiver control). These boards are further grouped into "register maps", which represent data from related subsystems (an antenna, say, or the correlator).

The list of register maps is given below. Clicking on any one will take you to the list of boards for that register map (if several register maps are identical, only the first will be listed). Clicking on a board will take you to the list of register blocks for that board, each listed using the following columns: 

    registerMap.board.block[valid indices]       (data type) (archive status) (integration status)

The data type is one of the following:

  • boolean - A single-byte type, consisting of a 0 (false) or 1 (true), used for storing truth values
  • unsigned char - A single-byte type, used for storing ASCII characters, or small unsigned integers (0 - 255)
  • char - A single-byte type, used for storing ASCII characters, or small signed integers (0 - ± 127)
  • unsigned short - A two-byte type, used for storing integers (0 - 65535)
  • short - A two-byte type, used for storing signed integers (0 - ± 32767)
  • unsigned integer - A four-byte type, used for storing unsigned integers (0 - 4.29497e+09)
  • integer - A four-byte type, used for storing signed integers (0 - ± 2.14748e+09)
  • float - A four-byte type, used for storing floating-point values
  • double - An 8-byte type, used for storing floating-point values
  • date - An 8-byte type, used for storing high-precision dates. Registers of this type can be displayed by appending the following extensions to the register name:
    • (none) - a fractional MJD
    • .date - a UTC date string
    • .time - a time string
  • complex float - An 8-byte type, used for storing complex (re, im) floating-point values Registers of this type can be displayed by appending the following extensions to the register name:
    • (none) - a complex number
    • .real - the real part only
    • .imag - the imaginary part only
    • .amp - the amplitude (magnitude) of the complex number
    • .phase - the phase (in degrees) of the complex number

The archive status indicates which registers are archived:

  • archived - The register is archived
  • not archived - The register is not archived

The integration status indicates how the register is treated when frames are combined before being written to disk:

  • summed - The register is simply summed on integration
  • unioned - The register is bit-wise unioned on integration
  • pre-averaged - The register is averaged before being written to disk
  • post-averaged - The register is summed before being written to disk, and divided by the number of frames which were combined on read-in
  • not integrated - Only the last value of the register is archived
