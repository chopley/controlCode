setEncoderPosition rx, position

Set the caltert index mark corresponding to a given receiver.

Arguments:
RxBand rx
The receiver to which this position applies.
TertPos position
The encoder position to associate with this receiver.
Example:
The following example sets the encoder positions for the 30 GHz receiver to 19500: 
setEncoderPosition rx30ghz, 19500
Context:
The Caltert (calibration/tertiary) CAN module maintains a trio of encoder positions in XAC memory. These positions are meant to be read from 1-wire devices on the modules, but in practice this has proven to be flaky at best. Quite often the 1-wires are not found on startup, and as a result the encoder positions are not read.

Without valid encoder positions for the receiver positions, the module will obviously not respond to requests to move to predefined positions, nor will it allow the calibrator load to be moved.

This command allows the user to explictly tell the caltert module which encoder position to associate with a receiver.
