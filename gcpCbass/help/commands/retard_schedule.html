retard_schedule index number

Move a schedule one or more entries towards the back of the schedule queue.

Arguments:
QueueEntry index
The queue index of the schedule to be moved.
Count number
The number of queue positions to advance the schedule.

Example:
The following example moves entry 0 in the queue of pending schedules to entry 2, allowing the schedule that previously was in entry 1 to advance to entry 0. 
  retard_schedule 0, 2
Context:
The schedule command queues scheduling scripts to be executed in the order that they are queued. If it then turns out that some of the scripts in the queue need to be run before a particular script that is nearer to the front of the queue, then the retard_schedule command can be used to move it one or more entries further back in the queue. Note that this can also be done interactively in the topmost pane of the main viewer window.
