cd pathname

Change the current working directory.

Arguments:
Dir pathname
The full name of the directory that is to be used as the new working directory.

Example:
The following example changes the current working directory to a directory that contains scheduling scripts, then uses the scheduling script to queue one of these scripts. Without the preceding cd command, the user would have to type in the full path name of the observe.sch script. 
  cd ~bicepdaq/BICEP/sch
  schedule observe.sch

Context:
The cd and pwd commands act essentially like the unix shell commands of the same name.

When the user doesn't specify a full pathname to a command like schedule, the full pathname is constructed by appending the user's relative pathname to the current working directory. The cd command allows this working directory to be changed, while the pwd command lets one see what it is currently set to.

Warning: In scripts, pathnames to commands like schedule are converted to full pathnames when the script is compiled, not when they are run, so cd commands are essentially useless in scripts, because they are executed long after all filenames have been set in stone. In fact the only good reason that I can think of for using a cd command in a script is in the the control program initialization file, where it can be used to set the working directory of interactive users.

The reason for expanding filenames during compilation is to allow the compiler to check that the corresponding files and directories exist before accepting scripts for execution.
