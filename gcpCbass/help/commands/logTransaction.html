logTransaction device, serial number, location, date, who, [comment=?]

Log a transaction.

Mandatory Arguments:
String device
A valid device name, defined in a transaction catalog which can be loaded using the loadTransaction command.
String serial number
A valid serial number for the device, defined in a transaction catalog which can be loaded using the loadTransaction command.
String location
A valid location for the device, defined in a transaction catalog which can be loaded using the loadTransaction command.
Date date
A valid date for this transaction.
String who
An identifier for the person logging the transaction.

Optional Arguments:
String comment
An optional comment. This need not be a quoted string, but note that a comma in an unquoted string will be interpreted as an extra argument

Example:
In the following example, we assume a catalog containing (at least) the following lines has been loaded: 
  DEVICE Receiver
  SERIAL sillyName1, sillyName2, sillyName3
  LOCATION ant0,ant1,ant2,ant3,ant4,ant5,ant6,ant7,ant8
Once this catalog is loaded, the following logs a valid transaction: 
 logTransaction Receiver, sillyName2, ant3, 10-oct-2004, me, comment="This is a test"
This will place the following lines in the current archived logfile: 
041011 23:27:08 **** Device: (Receiver: sillyName2) (ant3) (10-OCT-2004 00:00:00) Logged by: (me)
041011 23:27:08 **** This is a test
Context:
see loadTransaction
