loadTransaction filename, [clear=?]

Load transactions from a catalog file.

Mandatory Arguments:
InputFile filename
The name and directory of a readable transaction file.
Optional Arguments:
Boolean clear
If true, clear the transaction catalog first. If false, the catalog will be added to the catalog in memory.

Example:
The following example adds the device identifiers, serial numbers and valid locations defined in ~bicepdaq/BICEP/conf/trans.cat to the current list of known transaction identifiers. 
 loadTransaction ~bicepdaq/carma/bicep/array/conf/trans.cat
Context:
Transactions provide a facility for tracking hardware changes on the antennas. A transaction consists of a device, its serial number, and a location (or state), logged to the logfile in a standard format via the logTransaction command.

Before a transaction can be logged, it must be described to the control program via an entry in a transaction catalog. Transactions can then be referred to by name, and the control system will check that the location and serial number are valid for the given device.

The format of loadTransaction files:
Transaction catalogs are text files containing the descriptions of one or more transactions. Blank lines are also legal, as are comments. Comments are introduced with the # character and extend to the end of the line.

Transaction specifiers occur in groups, each group consisting of a line specifying a valid device name, followed by one or more lines specifying valid serial numbers for that device, and one or more lines specifying valid locations for that device. The following example would be valid transaction definitions, and demonstrate the supported syntax: 

 # This is a comment.

 #-----------------------------------------------------------------------
 # The following entry describes a device called "I/O Module", 
 #-----------------------------------------------------------------------

 DEVICE I/O Module

 #-----------------------------------------------------------------------
 # Tell the control system that this device can have serial numbers 
 # SN0012 or SN0013
 #-----------------------------------------------------------------------

 SERIAL SN0012, SN0013

 #-----------------------------------------------------------------------
 # Tell the control system that the ant0-ant7 are valid locations for
 # this device
 #-----------------------------------------------------------------------

 LOCATION ant0, ant1, ant2, ant3, ant4, ant5, ant6. ant7
