channelizer on|off

Temporarily disable or re-enable all of the IF amplifiers.

Arguments:
SwitchState state
The desired state of the IF amplifiers: on or off.

Example 1:
The following example turns off all of the IF amplifiers. 
 channelizer off
Example 2:
The next example re-enables those IF amplifiers that were switched on before the above command was executed. 
 channelizer on

Context:
The BICEP has 13 receivers (rx0..rx12) whose 10GHz bandpasses are split into 10 bands (band0..band9) by a channelizer. The resulting 130 signal paths are refered to as channels. Each of these channels contains an IF amplifier to boost the received signals before feeding them into one correlator module per band. The channelizer provides switches to enable or disable these IF amplifiers. Disabling an IF amplifier effectively disconnects the the input signal from its target correlator input and thus can be used to measure correlator offsets. To switch individual channels on or off, use the alternative channel command. To temporarily disable the channelizer by turning off all IF amplifiers, without loosing the record of which individual channels were turned on, use the channelizer command. When the same command is used to re-enable the channelizer, only those channels that were previously individually turned on are switched on. The remainder remain switched off.

Note that the channelizer is enabled when the system starts.
