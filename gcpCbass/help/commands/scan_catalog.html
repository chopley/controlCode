The scan_catalog command (usage)

Context:

The scan_catalog command loads the list of scans in filename into the control system. It must be executed before the scan command can be used.

Each entry in the scan catalog consits of a scan type, NORMAL or CONTINUOUS, a scan name, and the path to the scan file, separated by white space and terminated by a newline. Empty lines are ignored as are comments beginning with the # character. An alternate name for a scan can be defined by starting an entry with ALIAS followed by an existing scan name.

A CONTINUOUS scan contains the START, BODY and END sections allowing the body to be repeated, while a NORMAL scan does not.

Examples:

scan_catalog "$GCP_DIR/control/scan/scan.cat"
